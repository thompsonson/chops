# AtomicGuard Workflow Integration

This document describes how chops integrates with AtomicGuard's workflow engine for voice-triggered sysadmin operations.

**AtomicGuard repo:** `/home/mt/Projects/thompsonson/atomicguard/`
**MQTT contract (schemas, field tables, example payloads):** `docs/design/notes/conversation_id_design.md`

                                             ┌──────────────┐
                                             │  AtomicGuard  │
                                             │  MQTT Service │
                                             │              │

                                             │ workflow     │
                                             │ dispatch     │
                                             └──────┬───────┘
                                                    │
                              ┌──────────────────────┼──────────────────────┐
                              │                      │                      │
                              ▼                      ▼                      ▼
                    agent/intent/response   agent/workflow/events   agent/workflow/escalation
                              │                      │                      │
                              ▼                      ▼                      ▼
                       ┌──────────────┐       ┌──────────┐          ┌──────────┐

All responses carry `conversation_id` from the original request, enabling the UI to group messages into threads.

## MQTT Topics

| Topic | Direction | QoS | Purpose |
|-------|-----------|-----|---------|
| `agent/intent/request` | chops → AtomicGuard | 1 | JSON: `{text, conversation_id}` |
| `agent/intent/response` | AtomicGuard → chops | 1 | Intent classification result |
| `agent/workflow/events` | AtomicGuard → chops | 0 | Step-level progress updates |
| `agent/workflow/escalation` | AtomicGuard → chops | 1 | Must-deliver human-attention alerts |

**Full schemas with field tables and example payloads are maintained in the AtomicGuard repo:**
`docs/design/notes/conversation_id_design.md`

## Agent-Core (Relay Mode)

Agent-core acts as a transport layer. All transcriptions are forwarded to `agent/intent/request` — no local regex parsing.

**File:** `crates/agent-core/src/main.rs`

Flow:
1. Receive `voice/transcriptions` with `{text, is_final, conversation_id}`
2. Forward text + conversation_id to `agent/intent/request` (QoS 1)
3. Handle responses from AtomicGuard:
   - `agent/intent/response` → toast confirmation
   - `agent/workflow/events` → forward to UI via `agent/responses`
   - `agent/workflow/escalation` → error toast + forward to UI

## Conversation ID

The sender (chops) generates a `conversation_id` UUID for each voice command. AtomicGuard echoes it on every response, enabling the UI to group all messages from a single command into one conversation thread.

| ID | Generated by | Purpose |
|----|-------------|---------|
| `conversation_id` | Chops | Cross-system message correlation |
| `workflow_id` | AtomicGuard | Internal execution tracking |

A single `conversation_id` may map to multiple `workflow_id`s if AtomicGuard decomposes a command into multiple workflows.

## Available Workflows

These are pre-compiled task workflows in AtomicGuard's catalogue. Chops does not need to know their internal structure.

| Workflow | Voice triggers | What it does |
|----------|---------------|-------------|
| `health_check` | "check the system", "system status" | Runs `sysmon status`, `sysmon warn`, `systemctl --user list-units` |
| `disk_check` | "check disk", "disk space" | Runs `sysmon disk` |
| `resource_monitor` | "what's using memory", "top processes" | Runs `sysmon mem`, `sysmon proc` |
| `restart_service` | "restart chops web" | Restarts a systemd user service |
| `triage` | "something's wrong" | Parallel diagnostics + LLM summary |

New workflows are added to AtomicGuard's catalogue without any changes to chops. The embedding classifier routes to them automatically.

## What AtomicGuard Handles (chops does NOT implement)

- Intent classification (embedding classifier, 29ms, 93% accuracy)
- Workflow selection and parameter extraction
- Workflow execution (command generation, effector execution, guard validation)
- Retry logic (stagnation detection, backtracking)
- Undo/compensation for failed non-idempotent operations
- Human-readable summaries for workflow results

Chops is the **voice interface and notification layer**. AtomicGuard is the **workflow engine**.
